### 2.1 `percentile_calculator.py` — 分位计算器

**总体评价：✅ 可接受，需微调**

#### 优点
- 支持 rank 和 CDF 两种计算方法
- 包含置信度计算，边界情况处理完善
- 支持批量多指标分位计算

#### 问题 1：「最低数据点要求」与设计文档不符
- **现状**：`min_data_points = 52`（52 周 ≈ 1 年）
- **设计文档要求**：近 10 年历史数据
- **风险**：1 年数据计算的分位不具有统计意义，历史极值区间（0-20%）和（80-100%）无法有效识别
- **建议**：将默认值改为 `min_data_points = 520`（10 年 × 52 周），或改为按年数计算而非按数据点

```python
# 建议修改
def __init__(self, history_years: int = 10, min_data_points: int = 520):
    # 10年 × 52周 = 520，1年数据点无法支撑分位计算
```

#### 问题 2：缺少「数据不足时动态调整窗口」机制
- 设计文档提到"历史数据不足时动态调整分位计算窗口"，但代码未实现
- 建议增加：当数据点不足 520 时，可按实际数据点计算分位，但需在结果中标记 `confidence = 0` 或降低置信度

### 2.2 `divergence_analyzer.py` — 背离分析引擎

**总体评价：⚠️ 逻辑偏离设计，需较大改动**

#### 问题 1：背离类型命名与设计文档不一致

| 代码实现 | 设计文档 | 问题 |
|---------|---------|------|
| `BULLISH`（正向背离）| 逆向买点背离 | **方向相反** |
| `BEARISH`（负向背离）| 逆向卖点背离 | **方向相反** |

**详细分析**：
- **设计文档定义**：
  - 逆向买点背离 = 基本面跌至历史最差，但股价未同步创新低
  - 逆向卖点背离 = 基本面升至历史最优，但股价持续走弱
- **代码实现**：`BULLISH` 定义为「价格跌 + 估值升」，但未考虑基本面维度
- **核心缺失**：代码只检测「价格-估值」背离，缺少「基本面（景气度）- 价格」背离，这是设计文档的核心逻辑

#### 问题 2：缺少「边际改善」条件校验
- 设计文档要求买入信号需满足「景气边际改善：当期景气环比降幅收窄或小幅转正」
- 代码 `analyze_price_valuation_divergence()` 和 `analyze_price_macro_divergence()` 均未实现此逻辑

#### 问题 3：多维背离缺少「核心背离条件」实现
- 设计文档 3.2 条明确规定核心背离条件：
  - 买点：`价格分位 > 景气度分位`
  - 卖点：`景气度分位 ≥ 80%` 且近 3/12 月行业指数持续下跌
- 当前 `analyze_multi_dimensional_divergence()` 仅做简单信号投票，未校验核心背离条件

#### 建议修复方向
```python
# 建议新增方法（对齐设计文档）
def analyze_prosperity_price_divergence(
    self,
    prosperity_percentile: float,   # 景气度分位（核心）
    price_percentile: float,         # 价格分位（核心）
    prosperity_marginal_change: float,  # 边际改善指标（新增）
) -> Dict[str, Any]:
    """
    核心：基本面-价格背离检测（设计文档3.2条款）
    买点：prosperity_pct <= 20 且 price_pct > prosperity_pct
    """
    pass
```

### 2.3 `signal_judgment.py` — 信号判定器

**总体评价：⚠️ 判定逻辑偏离设计文档，需较大改动**

#### 问题 1：买入信号判定条件不符合设计文档

| 设计文档要求（3.3） | 代码实现 | 偏差 |
|------------------|---------|------|
| 景气度分位 ≤ 20% | 用 PE/PB 分位代替 | ❌ **未使用景气度** |
| 估值水位分位 ≤ 25% | 继承 PE/PB 分位 | ⚠️ 阈值 25%→20% 有差异 |
| 核心背离条件：价格分位 > 景气度分位 | 无此校验 | ❌ **完全缺失** |
| 景气边际改善 | 无此校验 | ❌ **完全缺失** |
| 风控前置校验 | 无此校验 | ❌ **完全缺失** |

#### 问题 2：卖出信号判定条件不符合设计文档

| 设计文档要求（3.3） | 代码实现 | 偏差 |
|------------------|---------|------|
| 景气度分位 ≥ 80% | 无此校验 | ❌ |
| 核心背离：景气持续创新高，但近3/12月行业指数持续下跌 | 无此校验 | ❌ |
| 估值条件：估值从历史高位持续回落 | 无此校验 | ❌ |

#### 问题 3：设计文档要求「全部条件同时满足」才触发买入，代码使用打分制
- **设计文档**：`AND` 逻辑（6 条全部满足才标记买入机会）
- **代码实现**：`signal_strength` 打分制（阈值 50 分触发买入）
- **风险**：打分制会误判，例如「估值低 + 无背离 + 无边际改善」也可能达到 50 分

#### 建议
将 `judge_buy_signal()` 改为严格 AND 逻辑：
```python
def judge_buy_signal_strict(
    self,
    prosperity_percentile: float,
    valuation_percentile: float,
    price_percentile: float,
    marginal_improvement: bool,
    risk_passed: bool,
) -> bool:
    """
    严格买入信号（设计文档3.2条款，全部条件同时满足）
    """
    conditions = [
        prosperity_percentile <= 20,           # 条件1
        valuation_percentile <= 25,            # 条件2
        price_percentile > prosperity_percentile,  # 条件3 核心背离
        marginal_improving,                     # 条件4
        risk_passed,                           # 条件5
    ]
    return all(conditions)
```

### 2.4 `risk_filter.py` — 风控过滤器

**总体评价：⚠️ 框架完整，但阈值和黑名单需对齐设计文档**

#### 问题 1：夕阳行业黑名单包含周期行业
```python
# 当前代码
self._sunset_industries = ['煤炭开采', '钢铁', '有色金属', '传统能源']
```
- **设计文档**：系统目标是分析煤炭、有色金属等强周期行业
- **问题**：黑名单将这些行业直接排除，导致核心分析对象无法进入逆向评分
- **建议**：清空或大幅缩减黑名单，仅保留真正永久性衰退行业（如教培）

#### 问题 2：产能过剩黑名单包含配置行业
```python
self._overcapacity_industries = ['钢铁', '水泥', '平板玻璃', '电解铝']
```
- 基础化工（系统目标行业之一）可能包含以上子行业，需评估是否会误伤

#### 问题 3：设计文档要求「排除长期政策永久压制、需求持续萎缩的夕阳行业」
- 当前实现依赖关键词匹配，不够精准
- 建议：黑名单改为精确匹配业务行业名称，而非模糊关键词

#### 建议保留的优点
- 多维度风控框架（夕阳/产能/政策/流动性/财务）设计合理
- `apply_all_filters()` 返回详细风控报告，便于后续分析

### 2.5 `reverse_scoring.py` — 逆向评分计算器

**总体评价：⚠️ 评分维度与设计文档不符**

#### 问题 1：缺少设计文档要求的「背离强度」得分维度
- **设计文档 5.1**：
  - 背离强度得分：最高 40 分
  - 基本面极值得分：最高 30 分
  - 估值安全边际得分：最高 20 分
  - 边际改善得分：最高 10 分
- **代码实现**：使用 valuation/divergence/trend/risk 四维度，缺少「基本面极值」维度

#### 问题 2：评分与信号判定逻辑重复
- `reverse_scoring.py` 和 `signal_judgment.py` 都包含估值分位计算
- 可能导致同一指标被重复评分两次

#### 问题 3：分位等级划分与设计文档不一致

| 分位区间 | 设计文档定义 | 代码实现 |
|---------|------------|---------|
| 0-20% | 极致底部 | 0-10%: 100分, 10-20%: 90分 |
| 20-80% | 中性震荡 | 20-30%: 75分, 30-50%: 50分 |
| 80-100% | 极致顶部 | 85-100%: 15分, >100%: 5分 |

#### 建议
按设计文档重新对齐评分权重：
```python
self._weights = {
    'divergence_strength': 40.0,  # 背离强度（设计文档最高40分）
    'prosperity_extreme': 30.0,  # 基本面极值（设计文档最高30分）
    'valuation_margin': 20.0,   # 估值安全边际（设计文档最高20分）
    'marginal_improvement': 10.0, # 边际改善（设计文档最高10分）
}
```

## 三、共性问题汇总

### 3.1 核心业务逻辑缺失（严重）

设计文档 3.2 条定义的**核心背离逻辑**在所有模块中均未正确实现：

> 逆向买点背离核心条件：**价格分位 > 景气度分位**
> 逆向卖点背离核心条件：**景气度分位 ≥ 80%** 且近 3/12 月指数持续下跌

当前代码只检测「价格-估值」背离，**完全遗漏了「基本面（景气度）- 价格」背离**，这是与设计文档最严重的偏离。

### 3.2 景气度数据未接入

- `RiskFilter`、`SignalJudgment`、`ReverseScoring` 均无 `prosperity_percentile` 参数
- 设计文档要求的「宏观景气指标（PMI/PPI/CPI）」未与行业分析流程打通
- `DataCollector.collect_macro()` 采集了宏观数据，但未传递给分析引擎

### 3.3 缺少「行业景气度」替代指标

设计文档 3.1 条提到景气度分位需量化，但系统无直接景气度数据源。需确认：
- 使用宏观指标（PMI 等）作为行业景气的代理指标？
- 还是使用行业指数点位作为景气代理？
